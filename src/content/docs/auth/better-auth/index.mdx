BetterAuth is a framework-agnostic TypeScript authentication library.

<ELI5>
BetterAuth is like building your own house instead of renting an apartment. You have full control over everything - your database, your UI, your rules. It takes more work to set up, but you own everything and there are no monthly bills.
</ELI5>

## Key Features

- **Open Source** - Free forever, MIT licensed
- **TypeScript First** - Full type safety
- **Database Agnostic** - Works with any database
- **Plugin System** - Add features as needed
- **No Vendor Lock-in** - Your data, your control

## Architecture

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Client    │────▶│   Server    │────▶│  Database   │
│  (React)    │     │  (auth.ts)  │     │  (Postgres) │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │
       │   createAuthClient()    betterAuth()
       │                   │
       ▼                   ▼
   auth-client.ts      lib/auth.ts
```

## Supported Databases

| Database | Adapter |
|----------|---------|
| PostgreSQL | Drizzle, Prisma, Kysely |
| MySQL | Drizzle, Prisma, Kysely |
| SQLite | Drizzle, Prisma, Kysely |
| MongoDB | Native adapter |
| Turso | LibSQL adapter |

## Available Plugins

| Plugin | Description |
|--------|-------------|
| Two Factor | TOTP-based 2FA |
| Magic Link | Passwordless email login |
| Passkey | WebAuthn/Passkey support |
| Organization | Multi-tenant teams |
| Username | Login with username |
| Anonymous | Guest sessions |

## Why BetterAuth?

### vs NextAuth/Auth.js

- More TypeScript-focused
- Cleaner API design
- Better plugin system
- Simpler database setup

### vs Clerk/Auth0

- Self-hosted (privacy)
- No monthly costs
- Full customization
- No vendor lock-in

### vs DIY Auth

- Battle-tested security
- Handles edge cases
- Active maintenance
- Community support
