WP Manager provides multiple ways to organize your WordPress sites beyond just servers and providers.

### Creating Projects

1. Navigate to **Projects** in the sidebar
2. Click **New Project**
3. Enter a name and optional description
4. Choose a color (10 preset colors)
5. Click **Create**

### Assigning Sites to Projects

1. Go to **Sites** and click on a site
2. Click **Edit** (or the pencil icon)
3. Select a project from the dropdown
4. Click **Save Changes**

Sites can belong to only one project at a time. The project is displayed on the site card and in the site detail view.

## Favorites

Mark frequently accessed sites as favorites. Favorite sites:
- Appear first in the sites list
- Show a star icon on their card
- Are sorted alphabetically among themselves

### Toggle Favorite

- Hover over a site card and click the star icon
- Or visit the site detail page and click the star

## Archive

Archive sites you no longer actively manage but want to keep for reference.

### Archiving Sites

1. Hover over a site card
2. Click the archive icon
3. The site moves to the Archived section

### Viewing Archived Sites

1. Go to **Sites** in the sidebar
2. Click **Archived** button in the header
3. View all archived sites

### Restoring Sites

From the Archived view, hover over a site and click the archive icon again to restore it to the active sites list.

## Global Search

Use the command palette (Cmd+K / Ctrl+K) to search across:

- Site names and URLs
- Server names
- Provider names
- Project names

### Keyboard Navigation

| Key | Action |
|-----|--------|
| `Cmd+K` / `Ctrl+K` | Open command palette |
| `↑` / `↓` | Navigate results |
| `Enter` | Go to selected item |
| `Escape` | Close palette |

Results are grouped by type (Sites, Servers, Providers, Projects) with icons for easy identification.

## CSV/JSON Export

Export your sites data for reporting, backup, or migration.

### Export Options

Navigate to **Sites** and click the **Export** button to download a CSV file.

**Included fields:**
- Site ID, name, URL
- Status (online/offline/unknown)
- WordPress version, PHP version
- SSL expiry date
- Server and provider info
- Project assignment
- Plugin/theme counts and update counts
- Favorite and archived status
- Notes

### API Endpoints

```bash
# CSV export (default)
GET /api/sites/export

# JSON export
GET /api/sites/export?format=json

# Include archived sites
GET /api/sites/export?includeArchived=true

# Include full IP addresses (masked by default)
GET /api/sites/export?includeIps=true
```

**Security note:** IP addresses are masked by default (e.g., `192.xxx.xxx.xxx`) to prevent accidental exposure. Use `includeIps=true` only when needed.

### Bulk Project Assignment

Organize multiple sites at once:

> Assign all sites with "shop" or "store" in the name to the E-commerce project

Claude generates bulk update logic:
```typescript
const ecommerceSites = await db.query.sites.findMany({
  where: or(
    like(sites.name, '%shop%'),
    like(sites.name, '%store%')
  )
})

await Promise.all(
  ecommerceSites.map(site =>
    db.update(sites)
      .set({ projectId: ecommerceProjectId })
      .where(eq(sites.id, site.id))

```

### Analyzing Project Distribution

Ask Claude for insights:

> Show me how many sites are in each project

Claude queries the database:
```typescript
const projectStats = await db
  .select({
    projectName: projects.name,
    projectColor: projects.color,
    siteCount: count(sites.id)
  })
  .from(projects)
  .leftJoin(sites, eq(projects.id, sites.projectId))
  .groupBy(projects.id, projects.name, projects.color)
  .orderBy(desc(count(sites.id)))
```

### Smart Archiving

Automate archiving based on criteria:

> Archive all sites that haven't been synced in 90 days and have no pending updates

Claude generates the archival logic:
```typescript
const staleDate = new Date(Date.now() - 90 * 24 * 60 * 60 * 1000)

const sitesToArchive = await db.query.sites.findMany({
  where: and(
    lt(sites.lastSyncedAt, staleDate.toISOString()),
    eq(sites.pendingPluginUpdates, 0),
    eq(sites.pendingThemeUpdates, 0),
    eq(sites.isArchived, false)
  )
})

await Promise.all(
  sitesToArchive.map(site =>
    db.update(sites)
      .set({ isArchived: true })
      .where(eq(sites.id, site.id))
  )
)
```

### Export Customization

Generate custom export formats:

> Export all e-commerce sites with only URL, WP version, and PHP version

Claude creates a custom export:

```typescript
const ecommerceSites = await db.query.sites.findMany({
  where: eq(sites.projectId, ecommerceProjectId),
  columns: {
    url: true,
    wpVersion: true,
    phpVersion: true
  }

const csv = ['URL,WordPress Version,PHP Version']
  .concat(
    ecommerceSites.map(site =>
      `${site.url},${site.wpVersion},${site.phpVersion}`
    )
  )
  .join('\n')
```
