## Using with Claude Code

Claude Code can help you maintain brand consistency and apply the style guide across documentation.

### Brand Consistency

**Apply brand colors:**
```
"Update this component to use JB Cloud brand colors from the style guide"
```

Claude Code will reference the StyleGuide component and apply:
- Primary colors
- Accent colors
- Background colors
- Text colors

**Check color usage:**
```
"Audit all components for brand color compliance and list any violations"
```

### Typography

**Apply typography scale:**
```
"Update these headings to match the style guide:
- H1: Hero titles
- H2: Section headings
- H3: Subsections
- Body: Paragraph text"
```

**Font loading optimization:**
```
"Optimize font loading for the brand fonts:
- Subset characters
- Preload critical fonts
- Add font-display swap
- Configure fallback fonts"
```

### Component Styling

**Style new components:**
```
"Create a button component matching the style guide with:
- Primary variant (brand blue)
- Secondary variant (neutral gray)
- Sizes: small, medium, large
- States: default, hover, active, disabled"
```

**Audit existing components:**
```
"Check if all UI components follow the style guide for:
- Border radius (8px default)
- Spacing scale
- Shadow system
- Transition timing"
```

### Documentation Design

**Apply design system to docs:**
```
"Update the documentation theme to match the brand:
- Header colors
- Sidebar styling
- Code block theme
- Link colors
- Button styles"
```

**Generate design tokens:**
```
"Extract design tokens from the style guide:
- CSS custom properties
- Tailwind config
- Styled-components theme
- Design tokens JSON"
```

### Real-World Examples

**Example 1: Rebrand components**
```
"The brand colors changed. Update all components from:
- Old primary: #3B82F6
- New primary: #8B5CF6

Find and replace across the codebase"
```

**Example 2: Dark mode support**
```
"Add dark mode variants for all brand colors:
- Light mode (existing)
- Dark mode (new)
- System preference detection
- Toggle component"
```

**Example 3: Export style guide**
```
"Export the style guide as:
- Figma design tokens
- CSS variables file
- Tailwind config
- Storybook theme

```
