The Vercel CLI lets you deploy, configure, and manage your projects from the terminal.

<ELI5>
The Vercel CLI is like having a remote control for your website. Instead of clicking around in a dashboard, you type commands to deploy your code, manage settings, and check on your site. It's faster for developers who live in the terminal.
</ELI5>

## Quick Start

```bash
# Login to your Vercel account
vercel login

# Deploy your project (preview)
vercel

# Deploy to production
vercel --prod

# Check who you're logged in as
vercel whoami
```

## Global Options

These flags work with any Vercel CLI command:

| Flag | Shorthand | Description |
|------|-----------|-------------|
| `--token` | `-t` | Use an auth token instead of login |
| `--scope` | `-S` | Execute in a different team scope |
| `--team` | `-T` | Specify team slug or ID |
| `--cwd` | - | Set working directory |
| `--debug` | `-d` | Show verbose output |
| `--no-color` | - | Disable colored output |
| `--help` | `-h` | Show command help |
| `--version` | `-v` | Show CLI version |

### Using Tokens for CI/CD

In automated environments, use a token instead of interactive login:

```bash
# Create token at vercel.com/account/tokens
vercel deploy --token $VERCEL_TOKEN
```

<ELI5 title="ELI5: Why use tokens?">
When a computer runs your deployment (like GitHub Actions), it can't click "Login with GitHub" like you can. A token is like a password that lets the computer prove it's allowed to deploy on your behalf.
</ELI5>

### Scoping to a Team

If you belong to multiple teams:

```bash
# List your teams
vercel switch

# Run command for specific team
vercel deploy --scope my-team

# Or use team ID
vercel deploy -T team_abc123
```

## Most Used Commands

| Command | What It Does |
|---------|--------------|
| [`vercel deploy`](/vercel/cli/deployment/) | Deploy your project |
| [`vercel dev`](/vercel/cli/utilities/) | Run local dev server |
| [`vercel env pull`](/vercel/cli/environment/) | Download env vars |
| [`vercel logs`](/vercel/cli/utilities/) | View runtime logs |
| [`vercel domains`](/vercel/cli/domains/) | Manage custom domains |

## Using with Claude Code

Ask Claude to run Vercel commands:

> "Deploy my project to production"

Claude will run `vercel --prod` and give you the deployment URL.

> "Show me the logs for my latest deployment"

Claude will run `vercel logs` with the appropriate flags.

> "Which Vercel team am I using?"

Claude will run `vercel whoami` and `vercel switch` to show your context.
